# iOS Assist Studio V3.0 IA / UX: Mobile Workstation

## 0. TL;DR

- V3.0 继续保留 V2.0 的工作站式信息架构：Dashboard、Device、Recorder、Workflow、Execute、Monitor。
- V3.0 不新增“设备池”或“团队管理”主导航。
- V3.0 需要新增跨页面概念：当前平台、当前设备资源锁、Target Library、Capability Report、Run Status / Test Result 分离。
- Target Library 可以先作为 Workflow 和 Recorder 的共享工作区能力，不急于升为独立 L1 导航。
- 所有主界面仍遵循 Summary First, Detail Later。

## 1. IA North Star

V3.0 的信息架构要把产品从：

```text
iOS local automation workstation
```

升级为：

```text
cross-platform mobile automation workstation
```

但仍然保持：

- 单人
- 单台当前设备
- 本地
- 可视化
- 可诊断
- 安全执行

## 2. Global Shell

### 2.1 Top Status Bar

V3.0 顶部状态栏展示：

- Platform：iOS / Android / 未连接
- Device：当前设备资源状态
- Driver：当前平台驱动状态
- Workflow：流程合法性
- Targets：目标库状态
- Execution：运行状态

状态语义：

- 就绪
- 提醒
- 错误
- 离线
- 运行中
- 暂停
- 占用中

禁止：

- 在状态栏展示 UDID、ADB serial、session、endpoint、source XML、logcat 原文或 syslog 原文。

### 2.2 Side Nav

主导航继续保持：

- L1 Dashboard
- L2 Device
- L3 Recorder
- L4 Workflow
- L5 Execute
- L6 Monitor

Target Library 不在第一阶段升为 L1。

原因：

- 目标库是跨 Device、Recorder、Workflow 的共享资产。
- 过早升为独立导航会让普通用户误以为“先建库才能自动化”。
- 更自然的入口是从设备画面创建目标、从录制动作创建目标、从 Workflow Inspector 选择目标。

后续如果目标库规模变大，可再评估独立导航。

### 2.3 Bottom Console

Bottom Console 增加平台相关只读分区摘要：

- 本机驱动
- 目标解析
- 平台日志
- 性能采样

仍然禁止：

- 主界面铺满技术日志。
- Console 触发设备动作。
- 展示未脱敏完整标识。

## 3. L1 Dashboard

Dashboard 目标：第一眼判断当前工作站是否能开始工作。

内容：

- 当前平台和当前设备摘要
- 当前设备资源锁状态
- Target Library 数量和问题数
- Workflow 合法性
- 今日运行、成功率、暂停率、基础设施错误率
- 最近运行的问题类型分布

交互：

- 点击设备摘要进入 Device
- 点击目标摘要进入 Workflow 的 Target Library 面板
- 点击运行问题进入 Monitor
- 点击 workflow 问题进入 Workflow Validate

边界：

- Dashboard 不编辑目标。
- Dashboard 不连接多台设备。
- Dashboard 不展示完整日志。

## 4. L2 Device

Device 是跨平台设备工作台。

### 4.1 Device Header

展示：

- 平台
- 设备摘要
- 系统版本
- 当前 App
- 驱动状态
- 资源锁状态

主操作：

- 连接设备
- 断开连接
- 截图刷新
- 启动 App
- 停止 App

辅助操作：

- 本机检查
- 复制脱敏诊断
- 高级指引

### 4.2 Remote Preview

预览区域能力：

- iOS：截图刷新 + WDA 动作
- Android：第一阶段可截图刷新 + 动作，后续可增强 scrcpy 流式远控
- 点击、滑动、输入、主页键或平台等价动作
- 从画面创建目标

交互规则：

- 运行中预览写操作锁定。
- 远控动作执行时自动持有短资源锁。
- 目标创建只写 Target Library，不执行设备动作。
- 无截图时引导用户刷新或连接。

### 4.3 Create Target From Device

入口：

- 选择画面区域
- 点选坐标
- 选择文字识别区域
- 读取当前 selector 候选

输出：

- 新建 target draft
- 打开 Target Inspector
- 保存前进行平台能力校验

## 5. L3 Recorder

Recorder 是把人工操作转成动作和目标的工作台。

页面结构：

- 录制控制区
- 当前设备预览
- 动作时间轴
- Target quick panel

时间轴展示：

- 动作类型
- 目标名称
- 等待策略
- 平台能力提示
- 是否有截图证据

默认隐藏：

- 坐标
- selector 原文
- OCR 原始文本
- 设备标识
- 本机路径

动作详情 Drawer：

- 可选择已有目标
- 可把坐标动作保存成 coordinate target
- 可把截图裁剪保存成 image target
- 可设置低置信策略：暂停、失败、跳过

## 6. L4 Workflow

Workflow 是 V3.0 的旗舰工作区。

### 6.1 Layout

建议结构：

```text
Workflow
  Left: Node Palette / Target Library switch
  Center: Canvas / Source / Validate
  Right: Node Inspector / Target Inspector
```

用户可见页签继续保持：

- 画布
- 源码
- 检查

### 6.2 Target Library Panel

Target Library 面板能力：

- 搜索目标
- 按平台过滤
- 按类型过滤
- 查看目标健康状态
- 拖目标到画布生成 Tap Target / Wait For Target 节点
- 打开 Target Inspector

目标健康状态：

- 可用
- 平台不支持
- 缺少截图
- 低置信
- 引用缺失

### 6.3 Node Inspector

节点 Inspector 新增：

- targetRef 选择器
- 当前目标摘要
- 平台 capability 提示
- 目标解析策略
- 失败策略
- 结果策略

用户文案：

- “目标”
- “等待目标出现”
- “找不到时”
- “低置信时”
- “作为断言”

避免直接展示：

- `targetRef`
- `platformRequirements`
- `selector`
- `confidenceThreshold`

这些字段只在 Source View 中可见。

### 6.4 Validate View

检查视图新增：

- 缺失目标
- 目标平台不匹配
- 当前平台不支持该节点
- OCR / 图像识别能力缺失
- appRef 缺失
- assertion 结果策略不完整

点击诊断：

- 能定位节点时选中节点。
- 能定位目标时打开 Target Inspector。
- 源码级问题切到 Source。

## 7. L5 Execute

Execute 是运行控制台。

### 7.1 Preflight

Preflight 新增：

- 当前设备资源空闲
- 当前平台 capability
- 当前 App 可启动
- Workflow 合法
- Target Library 合法
- 目标引用完整
- 目标解析能力可用

Preflight 不通过时：

- 主按钮禁用
- 展示短中文原因
- 提供跳转到 Device、Workflow 或 Target Inspector 的动作

### 7.2 Running Mode

运行中展示：

- 当前节点
- 当前目标
- 当前平台
- 当前动作
- 当前循环
- 目标解析置信度
- 最近截图 / 日志 / 性能摘要

暂停态：

- 低置信暂停
- 目标未找到暂停
- 手动检查暂停
- 系统弹窗暂停

暂停不等于失败。

### 7.3 Confirmation Modal

开始运行前确认：

- 平台
- 当前设备
- 当前 App
- Workflow
- 目标数量
- 模式和轮次
- 串行执行
- 安全停止
- 本地证据

不展示：

- 完整设备标识
- endpoint
- session
- 原始 capability JSON

## 8. L6 Monitor

Monitor 是本地运行复盘和问题归因中心。

### 8.1 Overview

新增 KPI：

- 任务完成率
- 用例通过率
- 暂停率
- 基础设施错误率
- 目标解析失败率
- 平均目标解析耗时

### 8.2 Run History

记录行展示：

- workflow 名称
- 平台
- 任务状态
- 用例结果
- 问题类型
- 耗时
- 目标问题数量

### 8.3 Run Detail

详情 Drawer 展示：

- Execution Path
- Target Resolution Chain
- Visual / OCR Evidence
- Logs
- Performance Samples
- Assertions
- Recommendations

结果表达：

- “任务已完成，用例通过”
- “任务已完成，断言失败”
- “任务暂停，等待人工确认”
- “任务失败，驱动未就绪”
- “任务停止，用户手动结束”

避免：

- 把所有非通过都显示为失败。
- 把设备问题算成业务断言失败。

## 9. Target Inspector UX

Target Inspector 是 V3.0 新的核心编辑器。

字段：

- 名称
- 类型
- 平台范围
- 识别策略
- 置信度
- 等待时间
- 失败策略
- 最近验证结果

动作：

- 从当前截图更新图片
- 测试识别
- 复制脱敏摘要
- 查看引用节点
- 删除未引用目标

删除规则：

- 被 workflow 引用的目标不能直接删除。
- 删除前必须展示引用节点。
- 删除不触发设备动作。

## 10. Empty States

V3.0 空态必须给下一步动作：

- 未连接设备：连接一台手机
- 未选择平台：连接设备后自动识别
- 无目标：从当前画面创建目标
- 目标无截图：刷新截图后重新裁剪
- 当前平台不支持：查看驱动能力
- 无运行记录：先去执行一次

空态不展示开发路线、竞品说明或内部阶段计划。

## 11. UX Boundaries

V3.0 必须继续遵守：

- 主界面短中文
- Summary First, Detail Later
- 技术细节进入 Drawer / Inspector / Advanced / Console
- 运行中锁定危险编辑
- 低置信暂停
- 安全停止
- 本地隐私
- 不做营销页

## 12. Verification UX

每个 V3.0 UI 能力需要可测试状态：

- 无设备
- iOS connected
- Android connected
- driver warning
- target invalid
- target low confidence
- running
- paused
- stopped
- infrastructure error
- assertion failed

Widget tests 应覆盖这些状态，不依赖真实设备。
