# iOS Assist Studio V4.0 PRD: Mobile Automation Workstation

## 0. TL;DR

- V4.0 定位为本地优先的移动自动化工作站，不是 SaaS、设备池、群控或团队测试管理平台。
- V4.0 从 V3.0 跨平台工作站继续升级，正式吸收 Airtest、Pyxelator、Appium Inspector 和 appium-mcp 的成熟价值。
- V4.0 不恢复 Node 中间层。Node 已退出主链路；若既有 Legacy Node 价值被覆盖，应进入删除计划。
- V4.0 是 Mobile First：iOS 和 Android 都必须是正式闭环。工程可以先用 iOS 真机深验证，但 4.0 首版必须让 Android 真机完成最小冒烟。
- V4.0 仍保持本地、单人、当前设备、串行执行、安全停止、Project DSL 真源和本地证据。

## 1. Product Positioning

V4.0 的产品定位：

```text
Enterprise-grade Local Mobile Automation Workstation
```

它面向不会 Appium、不会 WDA、不会编程的运营、QA 和自动化初学者，也服务需要深度 Inspector、可视化工作流、视觉定位和本地证据的自动化工程师。

产品体验目标：

- 像 Appium Inspector 一样能看清当前 App。
- 像 AirtestIDE 一样能录制、运行、产出报告。
- 像 Pyxelator 一样能用截图定位目标。
- 像 appium-mcp 一样为 AI 自动化协作预留受控工具接口。
- 像本地 IDE 一样管理 workflow、模板、证据和报告。

## 2. Product Principles

V4.0 遵循以下原则：

- Summary First：主界面先展示状态摘要，细节进入抽屉、Inspector、控制台或报告。
- Local First：运行历史、截图、模板、日志和报告都保存在本机。
- Mobile First：DSL、Runtime、Inspector、Vision 和 Evidence 都按 iOS / Android 双平台设计。
- Runtime First：所有设备动作必须经过 Runtime 状态机。
- Reuse First：能依赖就依赖，能适配就适配，能移植就移植，能复制小模块就复制，不能照搬时才自研。
- Safe First：视觉低置信、目标缺失、设备异常和系统限制都必须暂停、阻断或给出明确引导。

## 3. Users

Primary User：

- 不懂 Appium / WDA / ADB 的运营或 QA。
- 需要通过可视化方式录制、检查和运行移动自动化流程。

Secondary User：

- 自动化工程师。
- 需要 Appium session、元素树、视觉目标、日志、报告和 DSL 源码视图。

交互优先级：

```text
可理解性 > 稳定性 > 可扩展性 > 工程灵活性
```

## 4. Scope

V4.0 包含：

- Flutter Desktop Mac App 本地工作站。
- Dart Runtime 主编排。
- iOS Adapter：Appium / XCUITest / WDA。
- Android Adapter：Appium UiAutomator2 / ADB。
- Python Sidecar：Airtest / Pyxelator / 视觉能力。
- Project DSL：流程唯一真源。
- Inspector：截图、元素树、属性、命令调试。
- Target Library：坐标、元素、图片、区域、文本目标。
- Vision Core：模板定位、视觉断言、视觉分支、低置信暂停。
- Recorder：录制动作、截图模板、动作转流程。
- Workflow：画布、源码、校验、节点运行态。
- Execute：串行运行、安全停止、暂停收口。
- Monitor：本地历史、证据、失败归因、报告。
- AI / MCP Core：受控工具协议、生成建议、失败解释。

V4.0 不包含：

- 云端 SaaS。
- 多租户。
- 权限系统。
- 审计系统。
- 远程设备池。
- 多设备并发。
- 群控。
- 默认上传截图。
- 任意脚本执行。
- Node 中间 API、Node Runner 或 Node MCP 常驻服务。

## 5. Information Architecture

V4.0 一级模块：

| 模块 | 目标 |
|---|---|
| 总览 | 看到当前设备、流程、执行和风险摘要 |
| 设备 | 连接、驱动、会话、平台能力、真机预览 |
| 检查 | 截图、元素树、属性、命令和当前界面诊断 |
| 录制 | 捕获动作、裁剪模板、生成流程 |
| 目标 | 管理坐标、元素、图片、区域和文本目标 |
| 流程 | 可视化画布、源码视图、校验和节点编辑 |
| 视觉 | 模板库、识别测试、置信度和视觉证据 |
| 运行 | 单次、循环、持续、安全停止、暂停收口 |
| 记录 | 历史、截图、日志、失败归因和本地报告 |
| 助手 | AI 建议、流程草稿、失败解释和 MCP 工具入口 |

首版可按工程节奏逐步点亮模块，但信息架构必须保持这个终局结构。

## 7. Platform Acceptance

iOS Acceptance：

- 至少一台 iPhone 真机可连接。
- 可截图。
- 可获取 Inspector 基础信息。
- 可运行基础 workflow。
- 可生成本地证据。

Android Acceptance：

- 至少一台 Android 真机可发现。
- 可建立 Appium UiAutomator2 session。
- 可截图。
- 可执行 Tap / Wait / Swipe / Input 冒烟。
- 可产出本地运行记录。
- Android 失败时必须给出短中文诊断。

## 8. Privacy And Safety

- 不展示完整设备标识。
- 不展示完整 session。
- 不展示本机绝对路径。
- 不展示账号、证书主体或隐私截图内容。
- 截图证据默认滚动清理。
- AI 默认不上传截图。
- 复制诊断必须脱敏。
- Runtime 拥有所有设备动作。
- 视觉和 AI 都不能直接下发点击。

## 9. Relationship With V2.0 And V3.0

V2.0 提供 Flutter Mac App、Dart Runtime、Project DSL、画布、Recorder、Execute 和 Monitor 的本地工作站基础。

V3.0 提供跨平台 Runtime、Target Library、Android adapter 和结果模型方向。

V4.0 在 V3.0 基础上正式引入开源成熟能力融合：

- Airtest：视觉自动化、断言、报告思路。
- Pyxelator：截图模板定位。
- Appium Inspector：专业 Inspector 和 session 体验。
- appium-mcp：AI 工具协议和受控自动化入口。
