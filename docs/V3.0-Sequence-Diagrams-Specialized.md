# V3.0 Sequence Diagrams Specialized

本文件是 V3.0 时序图专项产物。它不是提示词，而是可直接渲染的 Mermaid 时序图真源。

共同约束：

- UI 不直接调用 iOS / Android adapter。
- Runtime 拥有资源锁、状态机、Workflow、Target 和 Evidence。
- Target Resolver 只返回解析结果和证据，不直接点击。
- 低置信不盲点，默认进入暂停或失败收口。
- 复制和展示内容必须脱敏。

## 1. Connect Current iOS Or Android Device

```mermaid
sequenceDiagram
  actor User as 用户
  participant UI as Flutter UI
  participant Runtime as Dart Runtime
  participant Lock as 资源锁
  participant Driver as Mobile Driver
  participant IOS as iOS Adapter
  participant Android as Android Adapter
  participant Device as 当前设备
  participant Snapshot as Runtime Snapshot

  User->>UI: 点击连接设备
  UI->>Runtime: connectCurrentDevice()
  Runtime->>Lock: 确认空闲
  Lock-->>Runtime: idle
  Runtime->>Driver: discoverCurrentDevice()
  Driver-->>Runtime: 平台和脱敏设备摘要

  alt iOS
    Runtime->>IOS: connect()
    IOS->>Device: 建立 Appium/WDA 会话
    Device-->>IOS: 会话就绪
    IOS-->>Runtime: capability report
  else Android
    Runtime->>Android: connect()
    Android->>Device: 建立 UiAutomator2/ADB 会话
    Device-->>Android: 会话就绪
    Android-->>Runtime: capability report
  end

  Runtime->>Driver: captureScreenshot()
  Driver->>Device: 获取截图
  Device-->>Driver: 当前画面
  Driver-->>Runtime: 截图引用
  Runtime->>Snapshot: 写入 platform/device/driver/targets
  Runtime-->>UI: 连接成功和当前画面
  UI-->>User: 显示设备画面
```

## 2. Create Target From Device Preview

```mermaid
sequenceDiagram
  actor User as 用户
  participant Preview as 设备预览
  participant Draft as 目标草稿
  participant Inspector as 目标 Inspector
  participant Runtime as Dart Runtime
  participant Lock as 资源锁
  participant Store as Target Store
  participant Validator as Target Validator
  participant Workflow as Workflow UI

  User->>Preview: 点击创建目标
  Preview->>Lock: 请求 targetCapture
  Lock-->>Preview: 已锁定目标创建
  User->>Preview: 点选或框选画面
  Preview->>Draft: 生成目标草稿
  Draft-->>Inspector: 打开编辑器
  Inspector-->>User: 建议名称和策略
  User->>Inspector: 保存目标
  Inspector->>Runtime: saveTarget(draft)
  Runtime->>Validator: 校验平台和字段

  alt 校验通过
    Validator-->>Runtime: ok
    Runtime->>Store: 写入目标
    Store-->>Runtime: targetId
    Runtime->>Lock: 释放 targetCapture
    Runtime-->>Inspector: 保存成功
    Inspector-->>Workflow: 目标可引用
  else 校验失败
    Validator-->>Runtime: 短中文问题
    Runtime->>Lock: 保持可编辑
    Runtime-->>Inspector: 展示原因和下一步
  end
```

## 3. Record Flow And Promote To Workflow

```mermaid
sequenceDiagram
  actor User as 用户
  participant Recorder as 录制页
  participant Preview as 设备预览
  participant Timeline as 动作时间轴
  participant Runtime as Dart Runtime
  participant Lock as 资源锁
  participant DSL as DSL Builder
  participant Validator as Workflow Validator
  participant Store as Workflow Store
  participant Workflow as 编辑流程

  User->>Recorder: 开始录制
  Recorder->>Runtime: startRecording()
  Runtime->>Lock: 请求 recording
  Lock-->>Runtime: 已锁定录制
  Runtime-->>Recorder: 录制中

  User->>Preview: 点击或选择目标
  Preview->>Timeline: 追加 Target + Action
  Timeline-->>Recorder: 展示用户可读动作
  User->>Timeline: 调整顺序和等待

  User->>Recorder: 生成流程
  Recorder->>Runtime: promote(actions)
  Runtime->>DSL: 构造 Project DSL
  DSL->>Validator: 校验结构和目标引用

  alt 校验通过
    Validator-->>DSL: ok
    DSL-->>Runtime: workflow
    Runtime->>Store: 保存当前流程
    Runtime->>Lock: 释放 recording
    Runtime-->>Workflow: 打开编辑流程
  else 校验失败
    Validator-->>Runtime: 诊断列表
    Runtime-->>Recorder: 定位问题动作
  end
```

## 4. Execute Tap Target Node

```mermaid
sequenceDiagram
  actor User as 用户
  participant Execute as 运行任务
  participant Runtime as Dart Runtime
  participant Lock as 资源锁
  participant Runner as Workflow Runner
  participant Resolver as Target Resolver
  participant Driver as Mobile Driver
  participant Device as 当前设备
  participant Evidence as Evidence Store
  participant Snapshot as Runtime Snapshot

  User->>Execute: 点击开始运行
  Execute->>Runtime: runWorkflow(config)
  Runtime->>Runner: preflight()
  Runner-->>Runtime: 设备/流程/目标可运行
  Runtime-->>Execute: 展示确认
  User->>Execute: 确认运行

  Runtime->>Lock: 请求 workflowExecution
  Lock-->>Runtime: 已锁定运行
  Runtime->>Runner: 执行 Tap Target
  Runner->>Resolver: resolve(targetRef)
  Resolver->>Driver: captureScreenshot/getPageSource
  Driver->>Device: 获取现场
  Device-->>Driver: 截图和可用源码
  Driver-->>Resolver: 现场数据
  Resolver-->>Runner: 位置/区域/置信度/证据
  Runner->>Evidence: 写入目标解析事件

  alt 置信度足够
    Runner->>Driver: tap(resolvedPoint)
    Driver->>Device: 执行点击
    Device-->>Driver: 动作完成
    Driver-->>Runner: action ok
    Runner->>Evidence: 写入动作事件
    Runner->>Snapshot: 更新当前节点
  else 置信度不足
    Runner->>Runtime: pause(reason)
    Runtime->>Evidence: 写入暂停证据
    Runtime-->>Execute: 展示暂停介入
  end
```

## 5. Low Confidence Pause And Manual Intervention

```mermaid
sequenceDiagram
  actor User as 用户
  participant Runtime as Dart Runtime
  participant Runner as Workflow Runner
  participant Resolver as Target Resolver
  participant Execute as 运行任务
  participant DeviceUI as 连接设备
  participant Inspector as 目标 Inspector
  participant Evidence as Evidence Store
  participant Lock as 资源锁

  Runner->>Resolver: resolve(targetRef)
  Resolver-->>Runner: low confidence
  Runner->>Runtime: enterPaused(target issue)
  Runtime->>Evidence: 写入低置信证据
  Runtime-->>Execute: 暂停，等待人工确认
  Execute-->>User: 显示原因、截图和建议

  User->>DeviceUI: 查看现场
  DeviceUI->>Runtime: requestPreview()
  Runtime-->>DeviceUI: 当前截图
  User->>Inspector: 更新目标区域或策略
  Inspector->>Runtime: saveTarget(update)
  Runtime-->>Inspector: 保存成功

  User->>Execute: 收口暂停
  Execute->>Runtime: resolvePause()
  Runtime->>Lock: 清理 paused 上下文
  Lock-->>Runtime: 回到 idle
  Runtime->>Evidence: 写入人工介入结果
  Runtime-->>Execute: 可重新运行或继续下一步
```

## 6. Monitor Opens Run Detail

```mermaid
sequenceDiagram
  actor User as 用户
  participant Monitor as 查看结果
  participant Runtime as Runtime Evidence API
  participant Evidence as Evidence Store
  participant Detail as Run Detail Drawer
  participant Target as Target Library
  participant Workflow as 编辑流程

  User->>Monitor: 选择一条运行记录
  Monitor->>Runtime: getRunDetail(runId)
  Runtime->>Evidence: 读取 metadata/events/screenshots/logs/performance
  Evidence-->>Runtime: 脱敏详情
  Runtime-->>Detail: 任务状态/用例结果/问题类型
  Detail-->>User: 展示运行详情

  User->>Detail: 查看目标解析链
  Detail-->>User: 展示截图、置信度和失败原因

  alt 目标问题
    User->>Detail: 修复目标
    Detail->>Target: 打开目标 Inspector
  else 流程问题
    User->>Detail: 修复流程
    Detail->>Workflow: 定位节点
  else 驱动问题
    User->>Detail: 复制脱敏诊断
    Detail-->>User: 摘要已复制
  end
```

## 7. Collect Logs And Performance During Run

```mermaid
sequenceDiagram
  participant Runner as Workflow Runner
  participant Driver as Mobile Driver
  participant IOS as iOS Adapter
  participant Android as Android Adapter
  participant Evidence as Evidence Store
  participant Monitor as 查看结果

  Runner->>Driver: collectLogs(window)
  alt iOS
    Driver->>IOS: collectLogs()
    IOS-->>Driver: syslog 摘要
  else Android
    Driver->>Android: collectLogs()
    Android-->>Driver: logcat 摘要
  end
  Driver-->>Runner: 脱敏日志切片
  Runner->>Evidence: 写入日志证据

  Runner->>Driver: samplePerformance()
  Driver-->>Runner: 性能采样摘要
  Runner->>Evidence: 写入性能证据
  Monitor->>Evidence: 读取 Run Detail
  Evidence-->>Monitor: 日志和性能摘要
```
