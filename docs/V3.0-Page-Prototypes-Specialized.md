# V3.0 Page Prototypes Specialized

本文件是 V3.0 页面原型专项产物。它不是提示词，而是低保真页面说明和 ASCII wireframe 真源。

设计基线：

- 风格：Native Quiet Mobile Workstation。
- 默认跟随 macOS 明暗模式，不强制深色。
- 任务导航：开始、连接设备、开始录制、编辑流程、运行任务、查看结果。
- Summary First, Detail Later。
- 技术细节进入 Drawer、Inspector、Advanced 或 Bottom Console。
- 主界面不展示完整设备标识、session、endpoint、source XML、ADB serial 或原始日志。

## 1. Global Shell

页面目标：让用户始终知道当前状态、当前位置和下一步。

布局区域：

- Top status：项目、平台、设备、目标、流程、运行、命令入口。
- Sidebar：任务导航。
- Workspace：当前任务页面。
- Bottom Console：默认收起的诊断区。
- Drawer / Inspector：右侧详情。

主操作：由当前页面决定，全局不放抢焦点的大按钮。

状态：

- 无设备：顶部设备显示“未连接”，开始页主操作为“连接设备”。
- 运行中：顶部运行显示“正在运行”，危险编辑锁定。
- 暂停：顶部运行显示“等待人工确认”，提供跳转。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 项目  平台  设备  目标  流程  运行                         ⌘K  设置        │
├──────┬───────────────────────────────────────────────────────────────────────┤
│ 开始 │                                                               工作区  │
│ 连接 │                                                                       │
│ 录制 │                                                                       │
│ 流程 │                                                                       │
│ 运行 │                                                                       │
│ 结果 │                                                                       │
├──────┴───────────────────────────────────────────────────────────────────────┤
│ 诊断 · 最近事件摘要                                         展开 Console     │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 2. 开始

页面目标：打开产品后立即知道现在能否开始，以及最合理下一步。

关键组件：

- Next Best Action。
- 当前设备、当前流程、最近结果三段摘要。
- 最近流程列表。
- 最近问题列表。

主操作：

- 未连接：连接设备。
- 可运行：开始运行。
- 有问题：定位问题。

右侧 Drawer：

- 点击摘要项打开只读详情，不执行设备动作。

Bottom Console：

- 只显示最近一条脱敏事件摘要。

状态：

- 空态：没有设备时只给“连接设备”。
- 错误态：显示问题类别和下一步，不展示日志。
- 运行中锁定态：主操作变成“查看运行”。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 今天                                                                       │
│ ┌────────────────────────────────────────────────────────────────────────┐ │
│ │ 还不能开始：没有连接设备                              [连接设备]       │ │
│ │ 连接后可以创建目标、录制流程并运行。                                  │ │
│ └────────────────────────────────────────────────────────────────────────┘ │
│ ┌────────────────────┐ ┌────────────────────┐ ┌───────────────────────┐ │
│ │ 当前设备 未连接    │ │ 当前流程 6 节点     │ │ 最近结果 2 次暂停     │ │
│ └────────────────────┘ └────────────────────┘ └───────────────────────┘ │
│ 最近流程                                  最近问题                         │
│ 登录检查       可运行        [编辑] [运行]  登录按钮低置信       [修复]     │
│ 下单流程       需修复目标    [查看]         驱动未就绪           [查看]     │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 3. 连接设备

页面目标：连接当前唯一设备，看到画面，并从现场创建目标。

关键组件：

- 当前设备摘要。
- 大型设备预览。
- 本机准备摘要。
- 目标创建入口。

主操作：

- 连接设备。
- 已连接后主操作变为“创建目标”或“开始录制”。

右侧 Drawer：

- 连接诊断。
- 高级指引。
- 目标创建草稿。

Bottom Console：

- Driver、Targets、Performance 分区提供脱敏细节。

状态：

- 空态：用数据线连接一台手机并解锁。
- 错误态：短原因和下一步，如“驱动未识别手机”。
- 运行中锁定态：预览写操作锁定，只能查看。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 连接设备                                      [刷新画面] [创建目标] [断开]  │

│                          │                                                   │
│ 下一步：开始录制         │                                                   │
├──────────────────────────┴───────────────────────────────────────────────────┤
│ 本机准备：全部就绪 · 最近截图：刚刚 · 运行：空闲                            │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 4. Create Target Drawer / Target Inspector

页面目标：把设备画面上的 UI 位置变成可复用目标。

关键组件：

- 名称。
- 类型建议。
- 平台范围。
- 识别策略。
- 最近验证。
- 引用节点。

主操作：

- 保存目标。
- 测试识别。

右侧 Drawer：

- Target Inspector 即 Drawer 本体。
- 高级区显示 selector、OCR、阈值、fallback。

Bottom Console：

- Target Resolver 证据摘要。

状态：

- 空态：先在设备画面点选或框选。
- 错误态：显示缺少截图、平台不支持或低置信。
- 运行中锁定态：目标定义不可保存。

```text
┌──────────────────────────────────────────┐
│ 目标                                      │
│ 名称          登录按钮                    │
│ 类型          图片 + 坐标建议             │
│ 平台          iOS / Android               │
│ 策略          先找图像，失败再暂停         │
│ 最近验证      可识别 · 92%                │
│ 引用节点      登录检查 / 点击登录          │
│                                          │
│ [测试识别]                    [保存目标] │
│                                          │
│ 高级                                          │
│ selector、OCR、阈值和 fallback 默认收起   │
└──────────────────────────────────────────┘
```

## 5. 开始录制

页面目标：让用户不用理解坐标，也能把人工操作转成 Target + Action。

关键组件：

- 录制控制。
- 设备预览。
- 动作时间轴。
- 动作详情 Drawer。

主操作：

- 开始录制。
- 录制中变为“停止录制”。
- 有动作后显示“生成流程”。

右侧 Drawer：

- 动作详情。
- 目标绑定。
- 等待策略。

Bottom Console：

- 只记录录制事件摘要，不显示原始 payload。

状态：

- 空态：连接设备并刷新画面。
- 错误态：设备或截图不可用。
- 运行中锁定态：不可录制。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 开始录制                           [开始录制] [生成流程]                   │
├──────────────────────────────────────┬───────────────────────────────────────┤
│                                      │ 动作时间轴                            │
│               设备画面               │ 1 点击 登录按钮                       │
│                                      │ 2 等待 首页出现                       │
│       点击画面或选择已有目标          │ 3 输入 手机号字段                     │
│                                      │ 4 点击 下一步                         │
├──────────────────────────────────────┴───────────────────────────────────────┤
│ 录制状态：空闲 · 可截图 · 运行未占用                                         │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 6. 编辑流程: Canvas + Target Library

页面目标：把流程维护成用户可读、可验证、可运行的自动化。

关键组件：

- 画布 / 源码 / 检查。
- Node Palette / Target Library 切换。
- Node Inspector / Target Inspector。
- Canvas overview。

主操作：

- 去运行。
- 保存更改只在有草稿时出现。

右侧 Drawer / Inspector：

- 节点属性。
- 目标引用。
- 平台能力提示。
- 找不到时和低置信时策略。

Bottom Console：

- 检查、目标解析、运行事件的脱敏摘要。

状态：

- 空态：从模板开始或从录制生成。
- 错误态：画布节点和 Inspector 同步展示问题。
- 运行中锁定态：画布只读。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 编辑流程                         画布  源码  检查              [去运行]     │
├──────────────────┬──────────────────────────────────┬──────────────────────┤
│ 节点 / 目标       │                                  │ Inspector            │

│ 商品卡片          │                                  │ 最近验证：可用       │

└──────────────────┴──────────────────────────────────┴──────────────────────┘
```

## 7. Workflow Source / Validate

页面目标：给工程用户精确编辑和诊断入口，但不让普通用户默认面对源码。

关键组件：

- Source JSON editor。
- 诊断列表。
- 节点和字段定位。
- 检查视图的用户化文案。

主操作：

- 保存源码。
- 重新检查。

右侧 Inspector：

- 当前诊断详情。
- 关联节点或目标。

Bottom Console：

- Validator 原始摘要脱敏显示。

状态：

- 空态：没有诊断时展示“流程可运行”。
- 错误态：短中文原因优先。
- 运行中锁定态：源码只读。

- 运行确认。

- 空态：先选择或修复流程。
- 错误态：预检失败并提供跳转。
- 运行中锁定态：配置和流程编辑锁定。

- 搜索框。
- 最近命令。

