# V3.0 ChatGPT Flow, Sequence And Prototype Brief

## 0. How To Use This Document

这份文档用于把 iOS Assist Studio V2.0 现有能力和 V3.0 目标能力一次性喂给 ChatGPT，让它生成：

- V3.0 产品总流程图
- V3.0 核心用户旅程图
- V3.0 跨平台 Runtime 架构图
- V3.0 关键时序图
- V3.0 各页面低保真原型图
- V3.0 页面状态、空态和错误态说明

推荐用法：

1. 若任务涉及产品体验、页面原型、HTML 或视觉方向，先读取 `docs/V3.0-Enterprise-Design-Master-Brief.md`。
2. 先复制“总控提示词”给 ChatGPT。
3. 再按需要追加“流程图提示词”“时序图提示词”或“页面原型提示词”。
4. 如果 ChatGPT 支持 Mermaid，让它优先输出 Mermaid。
5. 如果 ChatGPT 支持图片生成或 HTML/CSS，让它额外输出低保真页面图或 HTML 原型。

V3.0 原型生成时必须遵守体验优先级：

```text
UX > Consistency > Maintainability > Development Efficiency > Backward Compatibility
```

不得因为实现简单、沿用 V2.0 Tech Noir 或继续使用 Dashboard/Console 心智而牺牲用户体验。

## 1. Product Summary

项目名当前仍为 `iOS Assist Studio`。

V2.0 是本地 iPhone 自动化工作站：

- Flutter Desktop Mac App
- Dart Runtime
- Appium / WebDriverAgent / XCUITest
- 单台 USB iPhone
- Project DSL
- Recorder
- Workflow Canvas
- Execute
- Monitor
- 本地证据和诊断

V3.0 方向是单人单设备的跨平台移动自动化工作站：

- 本地 Mac 桌面 App
- 当前只连接一台 iOS 或 Android 设备
- 不做 SaaS
- 不做多用户
- 不做设备池
- 不做多设备并发
- 不做云端协作
- 不做完整测试管理平台

V3.0 要吸收 TestHub 类竞品解决过的痛点：

- 设备是资源，不是配置
- 远控、录制、执行要共享同一套设备资源锁
- 测试场景要可视化编排
- APP UI 不标准，需要目标库、图片、坐标、区域、OCR 和 selector 多策略
- 任务状态和用例结果要分离
- 执行过程要有截图、日志、性能和目标解析证据

## 4. V3.0 User Journeys

### 4.1 Journey A: First Connect Current Device

1. 用户打开 App。
2. Dashboard 显示未连接。
3. 用户进入 Device 或点击全局连接入口。
4. Runtime 检查本机依赖。
5. Runtime 识别当前唯一 iOS 或 Android 设备。
6. Runtime 选择 iOS adapter 或 Android adapter。
7. adapter 建立驱动会话。
8. Runtime 写入 snapshot。
9. Device Preview 显示当前截图。
10. Dashboard / Top Status 显示 ready。

### 4.2 Journey B: Create Target From Device Preview

1. 用户在 Device 预览中点击“创建目标”。
2. 用户选择坐标、区域、图片裁剪、文本区域或 selector。
3. UI 打开 Target Inspector。
4. 用户命名目标并设置策略。
5. Target validator 校验平台能力和字段。
6. Runtime 写入 Target Store。
7. Top Status 的 Targets 状态更新。
8. Workflow / Recorder 可引用该目标。

### 4.3 Journey C: Record A Flow And Promote To Workflow

1. 用户进入 Recorder。
2. Recorder 确认设备 ready、运行 idle、截图可用。
3. 用户开始录制。
4. 用户点击预览或选择目标生成动作。
5. 时间轴显示 Target + Action。
6. 用户整理动作顺序和等待。
7. 用户点击 Promote。
8. Runtime 构造 Project DSL。
9. Workflow validator 校验。
10. 保存为当前 workflow。
11. 用户进入 Workflow 或 Execute。

### 4.4 Journey D: Execute Target-Based Workflow

1. 用户进入 Execute。
2. Preflight 检查设备、驱动、workflow、targets、platform capability。
3. 用户点击运行。
4. Confirm Modal 展示平台、设备、workflow、目标数量、轮次、安全边界。
5. 用户确认。
6. Runtime 获取设备资源锁。
7. Runtime 按节点串行执行。
8. Tap Target 节点调用 Target Resolver。
9. Resolver 返回位置和置信度。
10. 置信度足够则 adapter 执行动作。
11. Runtime 写入事件和证据。
12. 结束后写入 Run Detail。

### 4.5 Journey E: Low Confidence Pause And Manual Intervention

1. Workflow 执行到 Wait For Target / Tap Target / Visual Branch。
2. Target Resolver 返回低置信或未找到。
3. Runtime 进入 paused。
4. Execute 显示暂停原因和目标证据。
5. 用户打开 Device Preview 查看现场。
6. 用户可更新目标、手动确认或停止。
7. 用户点击 Resolve Pause。
8. Runtime 收口 paused 到 idle 或按策略继续。
9. Monitor 记录 paused / inconclusive / target issue。

### 4.6 Journey F: Review Run Detail

1. 用户进入 Monitor。
2. 用户筛选 failed / paused / assertion failed / infrastructure error。
3. 用户打开 Run Detail。
4. 详情展示 Execution Path。
5. 展示 Target Resolution Chain。
6. 展示截图证据、日志摘要、性能采样。
7. 展示任务状态、用例结果、问题类型。
8. 用户复制脱敏诊断摘要。

## 6. Design Direction For Prototypes

视觉方向：

- Native Quiet Mobile Workstation
- macOS 原生桌面工具
- 默认跟随系统明暗模式，不强制深色工作台
- 高密度但低噪声
- 专业、简单、高效、可靠、舒服
- 类 Linear / Raycast / Finder / Shortcuts 的克制信息架构和原生交互
- 不做营销页
- 不做大面积装饰性渐变
- 不做 Tech Noir / Hacker / 科幻控制台
- 不做 Dashboard 堆数据
- 不做设备农场后台

布局约束：

- 桌面优先，建议 1440x900。
- 顶部状态栏 56px。
- 左侧导航 72px 收起态。
- Bottom Console 默认 48px，展开 320px。
- 主工作区按页面任务组织。
- 详情进入右侧 Drawer / Inspector。
- 主界面短中文。
- 技术字段进入 Advanced / Source / Console。

## 7. Total Prompt For ChatGPT

复制下面提示词给 ChatGPT：

```text
你是一名资深产品架构师、移动自动化平台产品经理和桌面端工具 UI/UX 设计师。

请基于以下产品上下文，为 iOS Assist Studio V3.0 设计完整的流程图、时序图和页面低保真原型。

产品背景：
- 当前 V2.0 是本地 iPhone 自动化工作站。
- V2.0 使用 Flutter Desktop Mac App + Dart Runtime + Appium / WebDriverAgent / XCUITest。
- V2.0 只支持单台 USB iPhone，Project DSL 是 workflow 唯一真源。
- V2.0 已有 Dashboard、Device、Recorder、Workflow、Execute、Monitor、Top Status Bar、Side Nav、Bottom Console、Global Command Center、Settings Drawer。
- V2.0 保持本地、单人、单设备、串行执行、安全停止、本地证据和隐私脱敏。

V3.0 目标：
- V3.0 是单人单设备的跨平台本地移动自动化工作站。
- 支持 iOS 或 Android 当前设备，但同一时刻只允许一台当前设备。
- 不做 SaaS、多用户、权限、审计、设备池、多设备并发、云端协作或完整测试管理平台。
- 吸收 TestHub 类竞品痛点：设备资源化、目标库、远控与自动化共存、图像/OCR/坐标/selector 多策略定位、执行状态与测试结果分离、运行证据可观测。
```
