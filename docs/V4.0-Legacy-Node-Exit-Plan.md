# iOS Assist Studio V4.0 Legacy Node Exit Plan

## 0. TL;DR

- V4.0 不恢复 Node 中间层。
- Legacy Node 只作为历史参考短期保留，不再承载主路径能力。
- 删除 Node 必须以能力覆盖为前提，不能删掉尚未迁移的唯一可用路径。
- 覆盖完成后，Legacy Node 进入物理删除，不继续留恋。

## 1. Exit Principles

Node 退出遵循：

- 不新增 Node API。
- 不新增 Node Runner。
- 不新增 Node MCP 常驻服务。
- Flutter / Dart 主路径不得调用 Node。
- appium-mcp 只吸收工具模型和权限边界，不吸收 Node 运行时。
- 删除前必须确认 Dart / Flutter / Python Sidecar 已覆盖对应价值。

## 2. Current Legacy Value

Legacy Node 仍可能保留的历史价值：

| 价值 | V4 替代方向 |
|---|---|
| 旧 sequence 初始化 | Project Config / Device Binding |
| 旧 A-F 点击执行 | Project DSL + Runtime workflow runner |
| 旧 Web 看板 | Flutter Mac App |
| 旧日志输出 | Bottom Console + Evidence Store |
| 旧截图录制 | Recorder + Device Preview |
| 旧 smoke 脚本 | Dart / Flutter tests + V4 boundary |

## 3. Exit Batches

### Stage 1: Guard

- `tool/v4_boundary_check.dart` 拦截 Flutter / Dart 主路径调用 Node。
- `npm run check` 指向 V4 boundary。
- 文档明确禁止恢复 Node 中间层。

### Stage 2: Cover

- Dart Runtime 覆盖连接、会话、截图、动作和运行。
- Project DSL 覆盖旧 sequence。
- Flutter Mac App 覆盖旧 Web 看板主入口。
- Python Sidecar 覆盖视觉能力，不借 Node 运行视觉。

### Stage 3: Disable

- Legacy npm 入口从主验证中移除。
- 文档只保留迁移参考，不再指导用户使用 Legacy。
- 新功能不得修改 Legacy Node。

### Stage 4: Delete

- 删除 `legacy/node`。
- 删除 Legacy npm scripts。
- 删除仅服务 Legacy 的 Node dependencies。
- 更新 README、AGENTS、AI_PROJECT_CONTEXT 和 docs index。
- 保留必要迁移说明或 ADR，不保留可运行 Node 主路径。

## 4. Delete Gate

进入物理删除前必须同时满足：

- V4 boundary 通过。
- V4 verify 通过。
- iOS 真机基础流程通过。
- Android 真机冒烟通过。
- 旧 A-F 模板可通过 Project DSL 导入和运行。
- Recorder 可生成 Project DSL。
- Monitor 可读取本地 evidence。
- 没有 Flutter / Dart 主路径引用 Legacy Node。

## 5. Stop Conditions

出现以下情况不得删除：

- 某项能力只有 Legacy Node 可用。
- Android 冒烟尚未完成。
- V4 boundary 不能检测 Node 回流。
- Python Sidecar 尚未有受控探测和失败兜底。
- 删除会破坏现有 Project DSL 导入或基础执行。

## 6. Acceptance

Node 完成退出时：

- 项目主路径没有 Node。
- Appium 仍可由 Dart Runtime 管理。
- Python 只作为视觉 Sidecar。
- appium-mcp 没有以 Node 服务进入主链路。
- 文档中不再把 Legacy Node 作为用户推荐路径。
